## Slide 3: What is a Database?
**Title:** Definition

- An **organized collection of data** stored electronically
- Like a **digital filing cabinet** — but smarter and faster
- **Three key functions:**
  - Organize data
  - Store data
  - Retrieve data efficiently

**Visual:** Filing cabinet icon → Database icon

---

## Slide 4: Why Can't We Just Use Excel?
**Title:** Why Do We Need Databases?

| Problem | Excel/Text Files | Databases |
|---------|-----------------|-----------|
| Data Integrity | ❌ Easy to make errors | ✅ Automatic validation |
| Concurrent Access | ❌ One user at a time | ✅ Thousands simultaneously |
| Security | ❌ Limited protection | ✅ Built-in access control |

---

## Slide 5: Three Main Reasons
**Title:** Why Databases?

1. **Data Integrity**
   - Ensures accuracy and consistency
   - Example: Bank transfers (deduct AND add)

2. **Concurrent Access**
   - Millions of users at the same time
   - Example: Double 11 Shopping Festival

3. **Security**
   - Access control and permissions
   - Example: Students view grades, teachers modify

---

## Slide 6: Basic Components
**Title:** What Makes Up a Database?

- **Tables** — Where data is stored (like spreadsheets)
- **Records (Rows)** — Individual entries
- **Fields (Columns)** — Categories of data
- **Queries** — Asking the database for information
- **Indexes** — Making searches faster

**Visual:** Simple table diagram with labels
